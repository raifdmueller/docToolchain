== Some Tips on how to use docToolchain in an Enterprise Environment

Some things are different when you use them at a larger scale.
Here are some tips from experience with docToolchain in enterprise environments.

=== Corporate Identity

Customize the microsite design so your employees can better identify with the documentation.
Take a look at xref:040_microsite/130_theming.adoc[] to see how to change the look and feel.

=== CI/CD Pipelines

To automatically build and deploy documentation, see the demo project: https://github.com/docToolchain/ci-cd-demo[ci-cd-demo].
It contains a https://github.com/docToolchain/ci-cd-demo/blob/main/.github/workflows/gh-pages.yml[GitHub workflow] and a https://github.com/docToolchain/ci-cd-demo/blob/main/.gitlab-ci.yml[.gitlab-ci.yml] as starters.

NOTE: GitHub and GitLab each provide only _one_ static page per repository.
Restrict the deploy pipeline to your main branch to avoid overwriting production docs from feature branches.

Set `DTC_HEADLESS=true` in your CI environment to skip interactive prompts.

=== Docker

The docToolchain Docker image is self-contained with all dependencies.
Run your build pipeline directly on this image — `dtcw4` detects it and uses the bundled tools.

[source, bash]
----
./dtcw4 docker generateSite
----
