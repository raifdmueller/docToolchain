= docToolchain v4 ŌĆö Getting Started

include::../_feedback.adoc[]

[NOTE]
====
docToolchain v4 is *experimental*.
It replaces Gradle with direct JVM execution ŌĆö no build tool at runtime, faster startup, fewer dependencies.
====

== What Changed in v4

[cols="1,1,1"]
|===
| | v3 (current stable) | v4 (experimental)

| *Runtime*
| Gradle + Groovy + Ruby (AsciiDoctor gem)
| Java only (AsciidoctorJ)

| *Wrapper*
| `dtcw`
| `dtcw4` (delegates to `dtcw` after install)

| *Startup time*
| ~10-15s (Gradle daemon)
| ~2-3s (direct JVM)

| *Ruby needed?*
| Yes (asciidoctor gem)
| No

| *Gradle needed?*
| Yes (at runtime)
| Only once at install time

| *Configuration*
| `docToolchainConfig.groovy` (unchanged)
| `docToolchainConfig.groovy` (unchanged)
|===

== Quick Start

=== 1. Install

Copy `dtcw4` and `dtcw` into your project directory, then:

[source,bash]
----
# Install Java (if not already available)
./dtcw local install java

# Install docToolchain v4 from source
./dtcw4 local install doctoolchain
----

This clones the `main-4.x` branch and builds the runtime JARs once (~2 min on first run).
After that, Gradle is no longer involved.

=== 2. Download a Template

[source,bash]
----
./dtcw4 local downloadTemplate
----

This downloads the arc42 template interactively.
For CI, use headless mode:

[source,bash]
----
./dtcw4 local downloadTemplate arc42 --lang=EN --help=plain
----

=== 3. Generate Output

[source,bash]
----
# HTML
./dtcw4 local generateHTML

# PDF
./dtcw4 local generatePDF

# Microsite (jBake)
./dtcw4 local generateSite
----

=== 4. List Available Tasks

[source,bash]
----
./dtcw4 tasks
----

== Available Tasks

[cols="1,2"]
|===
| Task | Description

| `generateHTML`
| Generate HTML5 from AsciiDoc using AsciidoctorJ (with diagram support)

| `generatePDF`
| Generate PDF using AsciidoctorJ PDF (with diagram support)

| `generateSite`
| Generate a microsite using jBake (same templates as v3)

| `downloadTemplate`
| Download arc42 or req42 templates (interactive or headless)

| `exportExcel`
| Export Excel sheets (.xlsx) to CSV and AsciiDoc tables

| `exportPPT`
| Export PowerPoint slides (.pptx) as PNG images with notes ŌĆö works on all platforms

| `publishToConfluence`
| Publish documentation to Confluence (same as v3)
|===

== Configuration

v4 uses the same `docToolchainConfig.groovy` as v3.
All existing configuration options continue to work.

=== New Options

[cols="1,1,2"]
|===
| Option | Default | Description

| `safeMode`
| `'UNSAFE'`
| AsciidoctorJ safe mode (`UNSAFE`, `SAFE`, `SERVER`, `SECURE`). Set to `SAFE` when processing untrusted content.

| `pdfThemeDir`
| `'./src/docs/pdfTheme'`
| Path to custom PDF theme directory (mapped to `pdf-themesdir`). Relative paths are resolved from the project root.
|===

== Updating

[source,bash]
----
./dtcw4 local install doctoolchain --update
----

This pulls the latest `main-4.x` and rebuilds the runtime JARs.

== Environment Variables

[cols="1,1,2"]
|===
| Variable | Default | Description

| `DTC_V4_BRANCH`
| `main-4.x`
| Git branch to install from. Override to test a feature branch.

| `DTC_HEADLESS`
| auto-detected
| Set to `true` for non-interactive mode (CI/CD).

| `DTC_SITETHEME`
| (none)
| URL to an external microsite theme zip.

| `DTC_TEMPLATE1..20`
| (none)
| URLs to custom documentation templates.
|===

== Differences from v3

=== AsciidoctorJ Instead of Ruby

v4 uses AsciidoctorJ (the Java port of AsciiDoctor) directly.
No Ruby gems, no external `asciidoctor` CLI needed.
Diagram support (PlantUML, ditaa, etc.) is included via `asciidoctorj-diagram`.

=== Cross-Platform PowerPoint Export

v3's `exportPPT` used Windows-only VBScript.
v4 uses Apache POI to render `.pptx` slides as PNG on any platform.
Legacy `.ppt` files are not supported ŌĆö only `.pptx`.

=== No Gradle at Runtime

After installation, all tasks run via `java -cp ... groovy.ui.GroovyMain scripts/<task>.groovy`.
No Gradle wrapper, no daemon, no `.gradle` cache.

== Migrating from v3

. Copy `dtcw4` into your project alongside `dtcw`
. Run `./dtcw4 local install doctoolchain`
. Replace `./dtcw` with `./dtcw4` in your build scripts
. Your `docToolchainConfig.groovy` works unchanged
. Test: `./dtcw4 local generateHTML`

v3 and v4 can coexist ŌĆö they install to separate directories (`~/.doctoolchain/docToolchain-3.x.x` vs `~/.doctoolchain/docToolchain-4.0.0`).

== Known Limitations

* Docker execution mode is not yet supported for v4
* SDKMAN installation is not yet available
* `exportEA` (Enterprise Architect) and `exportVisio` remain Windows-only in v3
* Some Gradle-based tasks (`htmlSanityCheck`, `exportStructurizr`) are not yet ported
