== Context and Scope

docToolchain sits between documentation authors and their audiences.
Authors write AsciiDoc sources in a version-controlled repository; docToolchain processes these sources and delivers the output to readers -- as HTML files, PDFs, microsites, or Confluence wiki pages.
Along the way it pulls data from external systems (Jira, Enterprise Architect, Git) to enrich the documentation.

=== Technical Context

[options="header",cols="1,2,2"]
|===
|External System |Protocol / Interface |Configuration

|Atlassian Confluence
|REST API v1 (Server/Data Center) or v2 (Cloud). HTTPS with Basic Auth or Bearer Token.
|`confluence.api`, `confluence.credentials`, `confluence.spaceKey` in config

|Atlassian Jira
|REST API v3 (Cloud) or v1 (Server, via `useV1Api` flag). HTTPS with Basic Auth or Bearer Token.
|`jira.api`, `jira.credentials`, `jira.exports[].jql` in config

|Enterprise Architect
|COM automation via VBScript. Reads `.eap` / `.eapx` project files directly.
|`exportEA.connection`, `exportEA.packageFilter` in config

|Structurizr
|File-based. Parses Structurizr DSL files, renders to PlantUML via `structurizr-export` library.
|`structurizr.workspace.filename` in config

|Git
|Local CLI (`git log`, `git shortlog`). No network access required.
|`changelog.dir`, `changelog.cmd` in config

|Excel / PowerPoint / Visio
|File-based. Apache POI for Excel (`.xlsx`), VBScript/COM for PowerPoint (`.pptx`) and Visio (`.vsdx`).
|`exportExcel` in config; PPT/Visio via script configuration

|AsciiDoctor JVM
|Java library (Gradle plugin `org.asciidoctor.gradle.jvm` 4.0.4). Processes AsciiDoc to HTML5, PDF, DocBook.
|`inputFiles`, `inputPath`, `outputPath` in config

|jBake
|Java library (Gradle plugin `org.jbake.site` 5.5.0). Generates static microsite from AsciiDoc + templates.
|`microsite.title`, `microsite.host`, `microsite.siteFolder` in config

|Pandoc
|External CLI tool. Converts between document formats (Markdown to AsciiDoc, DOCX to AsciiDoc).
|`pandoc` section in config
|===
