Quickstart

Generate your first documentation site in under five minutes — no global installation required.

🕑 4 min read📝 Updated Jun 2026🏷 v4.0

docToolchain follows the docs-as-code philosophy: your documentation lives next to your source code, written in plain text and built by a reproducible toolchain. This guide walks you through your very first build.

💡
Tip

Already have a project? You only need the dtcw wrapper — everything else is downloaded on demand.

Prerequisites #

The wrapper manages Java and all dependencies for you. You only need one of the following execution environments available on your machine:

  • local — installs into $HOME/.doctoolchain (recommended)
  • docker — runs everything inside a container
  • sdk — uses an SDKMAN-managed installation
ℹ️
Note

docToolchain requires Java 17. The local environment can install it for you with ./dtcw local install java.

Add the wrapper #

Drop the wrapper script into the root of your project and make it executable:

bash 
# download the wrapper
curl -Lo dtcw https://doctoolchain.org/dtcw
chmod +x dtcw

Generate the site #

Place your AsciiDoc sources under src/docs, then run a single command:

bash 
./dtcw local generateSite

› resolving environment …  ✓
› rendering AsciiDoc …     ✓
› building microsite …     ✓

BUILD SUCCESSFUL in 11s

Your generated site is written to build/microsite/output/. Open index.html to preview it locally.

⚠️
Warning

When using docToolchain as a Git submodule, set docDir to your actual documentation folder — pointing it at . can pull in the parent project's tasks.
